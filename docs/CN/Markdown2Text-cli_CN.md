# 📖 把经验文档变成“活”的文本指令  
**非开发者专用 · Agent 代运营 · 零代码变现你的知识**

这份指南是为你准备的——  
**如果你不太会写代码，但脑子里有宝贵的经验、独特的行业洞察，或者只是一套反复被问到的解决方案**，  
你可以把它们写成一个简单的 Markdown 文件，剩下的交给 Agent。  

Agent 会自动把这本“电子经验手册”变成一条 **可调用的文本指令**，任何人都能通过一行文字向它求助，而你可以按次收费、按心情分享，或者只是让自己的知识帮到更多人。

---

## 🌱 它能做什么

**传统困境**  
你的经验只能自己用，或者在群里反复回答同样的问题。

**现在**  
写成 `经验文档.md`，让 Agent 代你运营一条指令，像这样：

```
指令:我的经验;盆栽急救,绿萝,叶片发黄
```

Agent 会从你的文档里找到关于“绿萝”“叶片发黄”的描述，结合它的推理能力，给出精准的养护建议。

**你付出**：写下你知道的东西（Markdown 格式）。  
**你得到**：一条 24 小时在线的智能助手，别人每次使用都可能为你带来收益。

---

## 🧩 整体流程

1. **你写一份经验 MD**（稍后会教你怎么写最有效）。
2. **把文档发给 Agent**（比如通过聊天、邮件、或者放在一个可访问的链接）。
3. **Agent 自动注册指令**：它提取文档中的关键词、适用参数，生成一条标准指令。
4. **调用方发送指令** → Agent 找到你的文档，取出相关段落，加上调用方的问题，交给大模型推理。
5. **返回结果**——完全基于你的经验，但又能灵活适应每个提问者。

整个过程你不需要接触任何代码或服务器，Agent 就是你的“技能运营经理”。

---

## 📝 如何写好一份经验 MD

为了让 Agent 更好理解、更精准地调用，建议你的 Markdown 文件包含以下部分：

```markdown
# 盆栽常见问题急救手册

## 指令定义 (可放在文件开头，帮助 Agent 注册)
- 领域: 家庭园艺
- 动作: 盆栽急救
- 触发词: 盆栽, 叶子黄, 烂根, 浇水, 多肉, 绿萝, 红掌, 虎皮兰
- 参数: 植物名, 症状

## 经验内容

### 绿萝
#### 叶片发黄
- 原因：浇水过多或光照不足。
- 急救：立即停止浇水，移到散射光处，剪掉黄叶。
- 提醒：如果土壤仍有积水，可更换干燥的培养土。

#### 烂根
- 表现：根部变黑、变软，有异味。
- 处理：把植株脱盆，剪去腐烂根系，用多菌灵溶液浸泡 15 分钟，换新土重栽。

### 多肉植物
#### 化水
- 症状：叶片变透明、一碰就掉。
- 急救：停止浇水，加强通风，摘除化水叶片，必要时连根拔出晾干。
...
```

**关键点**：
- 文档标题尽量清晰，内容按**分类-症状-对策**结构化。
- 开头用简单文字描述你的领域、动作和常见触发词，Agent 会优先读取这些来生成指令元数据。
- 尽量多用具体关键词，这样 Agent 匹配更准。

> 不强制格式，但越结构化的内容，回答越贴近你的本意。

## 🤖 Agent Skill 配置（供技术人员/平台方）

> **已实现**：仓库 `text_cli/agent/CN/cli/nocode/markdown_converter.py` 是以下配置的 Python 标准库实现，零依赖。
> 参考 `docs/CN/Building_text-cli_guide_CN.md` §8.2 了解非代码模式的具体用法。

以下配置适用于 **openclaw** 或任何支持 Function Calling 的 Agent。  
你只需将此 Skill 添加到 Agent 中，它便具备了“经验文档代运营”的能力。

```json
{
  "name": "experience_directive_agent",
  "description": "将用户的 Markdown 经验文档转化为可调用的文本指令，并根据文档内容进行推理回答。",
  "system_prompt": "你是一个经验指令代运营助手。当用户上传 Markdown 格式的经验文档时，你需要解析文档并自动生成或更新对应的文本指令。当收到文本指令调用时，你需要从已存储的经验文档中找到最相关的内容，结合指令参数，调用大模型生成准确回答。\n\n指令解析：所有文本指令格式为“指令:<领域>;<动作>,<参数1>,<参数2>”。你要负责将其映射到对应的经验文档。",
  "tools": [
    {
      "name": "register_experience",
      "description": "注册或更新一份经验文档。接收 Markdown 文本或文件 URL，解析后存储，并生成/更新对应的指令元数据。",
      "parameters": {
        "type": "object",
        "properties": {
          "content": {
            "type": "string",
            "description": "Markdown 格式的经验文档全文，或可公开访问的文档 URL"
          }
        },
        "required": ["content"]
      }
    },
    {
      "name": "execute_experience_directive",
      "description": "执行一条基于经验文档的指令。传入完整的指令文本，Agent 会匹配已注册的经验，并结合指令中的参数进行推理。",
      "parameters": {
        "type": "object",
        "properties": {
          "directive": {
            "type": "string",
            "description": "完整的文本指令，例如：指令:家庭园艺;盆栽急救,绿萝,叶片发黄"
          }
        },
        "required": ["directive"]
      }
    }
  ],
  "tool_call_handler": {
    "register_experience": {
      "method": "POST",
      "url": "https://your-agent-backend.com/internal/register_experience",
      "description": "后端负责存储文档、生成指令映射"
    },
    "execute_experience_directive": {
      "method": "POST",
      "url": "https://your-agent-backend.com/internal/execute_directive",
      "description": "后端负责查询文档、拼接 prompt 并调用 LLM"
    }
  }
}
```

> **后端说明**：`register_experience` 和 `execute_experience_directive` 需要你（或你的平台）自行实现简单的存储与检索逻辑，可使用文件系统或轻量数据库，Agent 只是调用接口。如果希望完全免费启动，也可由 Agent 自身上下文来记忆文档，但受窗口限制。建议平台方提供以上接口。

## 🚀 完整演示（从零到一次调用）

### 第一步：非开发者小王上传经验

小王经营花店十年，他把经验整理成了 `盆栽急救手册.md`（内容如上），并在 Agent 对话框中说：

> 请帮我注册这份经验文档：  
> `https://example.com/potted_plant_firstaid.md`

Agent 调用 `register_experience`，解析文档中的“家庭园艺”领域、“盆栽急救”动作，生成指令模板：

```
指令:家庭园艺;盆栽急救,{植物},{症状}
```

### 第二步：调用方求助

远在北京的李先生家的绿萝叶片发黄，他对 Agent 说：

> “帮我问问那个盆栽急救指令，植物是绿萝，症状是叶片发黄。”

Agent 识别出关键词“盆栽急救”，组装完整指令：

```
指令:家庭园艺;盆栽急救,绿萝,叶片发黄
```

### 第三步：Agent 代运营推理

Agent 调用 `execute_experience_directive`，后端从文档中提取有关绿萝叶片发黄的段落，并生成以下 prompt 交给大模型：

```
你是一位资深园艺师，以下是经验文档中的相关内容：

【经验片段】
### 绿萝
#### 叶片发黄
- 原因：浇水过多或光照不足。
- 急救：立即停止浇水，移到散射光处，剪掉黄叶。
- 提醒：如果土壤仍有积水，可更换干燥的培养土。

现在用户提问：“绿萝叶片发黄怎么办？”请基于上述经验，给出简洁、体贴的建议。
```

大模型回答：

> 绿萝叶片发黄大多是水多或光照不够。先把花盆移到明亮但无直射光的位置，暂停浇水，把已经变黄的叶子剪掉。如果盆底还有积水，最好脱盆检查根系，换干爽的土。一般一周左右会慢慢恢复。

### 第四步：返回结果

该回答作为 `text-cli` 的标准响应返回给李先生：

```json
{
  "rst_types": "text",
  "rst_data": {
    "text": "绿萝叶片发黄大多是水多或光照不够。先把花盆移到明亮但无直射光的位置..."
  }
}
```

**全程小王没有写一行代码。他只提供了一份 md 文档。**

## 💰 如何变现？

你可以为这份经验指令设置一个 `Service Token`，并与调用方私下协商价格。  
Agent 运营方（或你自建的代理）可以在每次调用时检查 Token、计数、扣费。

当别人问你：“我的龟背竹叶边焦了怎么办？”  
你的指令就能立刻提供价值，而你获得相应的报酬。

## 🌟 进阶玩法

- **多文档融合**：Agent 可以同时注册多份经验（比如“家居风水”“旧书修复”），调用方通过 `领域;动作` 区分。
- **评论与迭代**：调用方可对回答打分，Agent 记录反馈，帮助你迭代文档。
- **社区市场**：未来，你可直接将经验指令的 Schema 发布到社区，让全世界的 Agent 都能发现你。

## 🔧 实际可运行

本指南描述的理念已通过仓库中的以下文件实现：

```
text_cli/agent/CN/cli/nocode/
├── markdown_converter.py    ← Markdown → 指令 转化引擎（186 行，零依赖）
└── 盆栽急救手册.md          ← 可直接运行的结构化经验文档示例
```

**一行启动**：

```bash
cd text_cli/agent/CN/cli/nocode
python markdown_converter.py 盆栽急救手册.md
```

启动后即可通过 `curl -X POST http://localhost:8000/cli/text_cli` 调用指令。
