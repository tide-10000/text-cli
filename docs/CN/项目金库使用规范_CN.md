## 序言

本文档定义 `text-cli` 项目金库的管理和使用规则。

项目金库是生态经济体系中 30% 日铸分流 + 捐赠 + 端点收入的汇聚池，由 lemondy 管理。金库的存在确保生态在算法自动化之外，保留必要的战略弹性。

本文档旨在确保这种弹性**不被滥用**——通过公开记录、用途限定和事后审计，让金库的每一笔收支都经得起全体协作者的审视。

---

## 一、金库资产构成

金库持有两类资产：

| 资产 | 来源 | 用途 |
|:---|:---|:---|
| **TCC（可用）** | 创世预捐 + 日铸 30% 分流 + 端点收入留存 + 社区捐赠 | 应急激励、战略加码、社区运营、开源收购 |
| **TCC（锁定）** | 金库可用 TCC 中划拨 | cTCC 兑换背书（1 TCC = 10,000 cTCC 背书额度） |
| **cTCC** | 端点回收 + 捐赠 | 社区激励发放、微小贡献即时奖励 |

---

## 二、支出审批

### 2.1 审批层级

| 支出类型 | 单次上限 | 审批要求 |
|:---|:---|:---|
| cTCC 兑换背书 | 无上限（受锁定 TCC 余额约束） | lemondy 确认周汇总 PR → 自动执行 |
| 应急激励 | ≤ 1 TCC / 次 | lemondy 裁量 + 广场公开理由 |
| 应急激励 | > 1 TCC / 次 | lemondy 裁量 + 广场公示 48h + 无有效异议 |
| 战略加码 | ≤ 2 TCC / 次 | lemondy 裁量 + 广场公开理由 |
| 战略加码 | > 2 TCC / 次 | 广场提案 → 48h 异议期 → lemondy 终裁 |
| 社区运营 | ≤ 0.5 TCC / 周 | 固定额度，周汇总 PR 中标注 |
| 开源收购 | 不限 | 按 `Ecological_economy_CN.md` §4.4 流程 |

### 2.2 紧急通道

安全漏洞修复等不可预见的紧急情况，lemondy 可绕过常规审批直接支出，但必须在 24h 内在广场公开理由和金额。

---

## 三、公示与审计

### 3.1 常规公示

- 金库余额在 `.bills/treasury/balance.md` 中实时更新
- 每次支出后更新 `.bills/treasury/expenditure.md`
- 每次收入后更新 `.bills/treasury/income.md`

### 3.2 季度报告

每季度由 AI 协作者（Tide）编制金库季度报告：

```
· 期初余额
· 本季收入（按来源分类）
· 本季支出（按用途分类）
· 期末余额（可用 / 锁定 / cTCC 储备）
· 异常支出说明（如有）
```

季度报告在广场发布，全体协作者可审查。

### 3.3 异议机制

- 任何协作者可对单笔支出提出异议
- 异议方式：在广场 `p_text-cli.md` 留言，附具体理由
- lemondy 在 72h 内回应
- 如异议被 ≥ 2 名合格投票者附议 → 触发金库支出重审

---

## 四、禁止事项

金库资产**不得**用于：

- lemondy 或任何协作者的个人消费
- 交易所上架 / 做市（文贝不进入公开交易所）
- 向非贡献者无偿赠予（除非社区激励明确标注为新贡献者欢迎礼）
- 任何未经广场公示的秘密支出

---

## 五、附则

- 本文档随生态经济体系文档同步修订
- 金库支出规则的重大变更需经全体协作者在广场达成共识
- 当前金库初始余额：5 TCC（lemondy 2026-05-04 创世预捐）
- 金库支出在 `.bills/treasury/` 中持续记录
